API Reference

Convert between Arabic numerals and Chinese capitalised numerals (人民币大写). Four endpoints, no auth, JSON only.

Overview

Base URL: http://localhost:8080 (development) — replace with your deployed origin.

Authentication: none. All endpoints are open. Rate limiting and API keys are not implemented in v1.

Content-Type: application/json for all requests and responses. UTF-8.

Error codes

All errors share one envelope: { "error": "<code>", "message": "<human readable>", "at": <optional offset> }.

HTTPcodeMeaning
400invalid_formatAmount does not match ^-?\d+(\.\d{1,2})?$, or JSON body malformed.
400out_of_rangeAbsolute value exceeds 999,999,999,999.99.
400unparsable_chineseChinese capitalised string could not be parsed; at field marks the offset where parsing failed.
413batch_too_largeBatch contains more than 200 entries.

Reverse parsing behaviour

The reverse and verify endpoints accept either currency unit (圆 or 元) and either terminator (整 or 正) interchangeably. The forward and batch endpoints always emit canonical 圆 + 整 form (PBOC ledger convention).

Negative amounts are supported across all endpoints: a leading - in the numeric form, a leading 负 in the Chinese form.

POST /api/convert

Convert an Arabic numeral amount to Chinese capitalised form.

Request

{
  "amount": "1234.56"
}

Response 200

{
  "chinese": "壹仟贰佰叁拾肆圆伍角陆分"
}

Code samples

curl -X POST http://localhost:8080/api/convert \
  -H 'Content-Type: application/json' \
  -d '{"amount":"1234.56"}'
import requests

r = requests.post(
    "http://localhost:8080/api/convert",
    json={"amount": "1234.56"},
)
print(r.json()["chinese"])  # 壹仟贰佰叁拾肆圆伍角陆分
const res = await fetch("http://localhost:8080/api/convert", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ amount: "1234.56" }),
});
const data = await res.json();
console.log(data.chinese); // 壹仟贰佰叁拾肆圆伍角陆分
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

func main() {
    body, _ := json.Marshal(map[string]string{"amount": "1234.56"})
    resp, _ := http.Post("http://localhost:8080/api/convert",
        "application/json", bytes.NewReader(body))
    defer resp.Body.Close()
    var out struct{ Chinese string `json:"chinese"` }
    json.NewDecoder(resp.Body).Decode(&out)
    fmt.Println(out.Chinese) // 壹仟贰佰叁拾肆圆伍角陆分
}

POST /api/convert/reverse

Parse a Chinese capitalised amount back into Arabic numerals. Accepts 圆/元 and 整/正 interchangeably.

Request

{ "chinese": "壹仟贰佰叁拾肆圆伍角陆分" }

Response 200

{ "amount": "1234.56" }

Error 400 (unparsable)

{
  "error": "unparsable_chinese",
  "message": "input does not match expected pattern",
  "at": 0
}

Sample (curl)

curl -X POST http://localhost:8080/api/convert/reverse \
  -H 'Content-Type: application/json' \
  -d '{"chinese":"壹仟元正"}'

Try it

POST /api/convert/verify

Compare a numeric amount and a Chinese capitalised string. Always returns 200 unless the numeric form itself is invalid.

Request

{
  "amount": "1234.56",
  "chinese": "壹仟贰佰叁拾肆圆伍角陆分"
}

Response 200 (match)

{ "match": true, "expected": "壹仟贰佰叁拾肆圆伍角陆分" }

Response 200 (mismatch)

{
  "match": false,
  "expected": "壹仟贰佰叁拾肆圆伍角陆分",
  "diffAt": 9,
  "message": ""
}

Try it

POST /api/convert/batch

Convert up to 200 amounts in one call. Per-item errors are reported inside the results array; the overall request still returns 200. Exceeding 200 entries returns 413.

Request

{
  "amounts": ["1234.56", "1000", "abc"]
}

Response 200

{
  "results": [
    { "amount": "1234.56", "chinese": "壹仟贰佰叁拾肆圆伍角陆分" },
    { "amount": "1000",    "chinese": "壹仟圆整" },
    { "amount": "abc",     "error": "invalid_format", "message": "amount must be a valid number with optional decimals" }
  ]
}

Error 413

{
  "error": "batch_too_large",
  "message": "batch size exceeds 200"
}
